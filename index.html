What is Web3 Staking?

Lido is a liquid staking protocol centered on Ethereum, where users can deposit ETH and receive stETH as a transferable tokenized position. The protocol mechanics are documented in the Lido token docs.

How Web3 Staking works

  1. Connect walletOpen the official Lido app and connect a supported Ethereum wallet.
  2. Choose ETH amountEnter the amount of ETH to stake and review the wallet transaction before signing.
  3. Receive stETHAfter staking, stETH represents the ETH supplied through the Lido protocol and can be held, transferred, or wrapped.
  4. Manage or withdrawUse the app to review rewards, wrap to wstETH, or start a withdrawal when needed.

Before you stake

Liquid position

Staking ETH through Lido mints stETH, which represents ETH supplied to the protocol while remaining usable across supported DeFi venues.

Transparent contracts

Core deployments are listed in the Lido deployed contracts documentation for domain and address checks.

Ethereum-native route

Lido operates around Ethereum proof-of-stake; the base staking model is explained in the Ethereum staking guide.

Web3 Staking FAQ

What is web3 staking on Lido?

It is an app route for ETH holders to use Lido's liquid staking protocol and receive stETH after staking ETH through the official interface.

Is web3 staking with Lido custodial?

Lido describes its protocol flow as non-custodial, but users still interact with smart contracts and should verify the domain, wallet permissions, and transaction details.

What token do I receive from web3 staking?

Staking ETH through Lido returns stETH; users may also wrap stETH into wstETH for integrations that prefer static balances.

Can I withdraw after web3 staking?

The Lido app includes a withdrawals flow; withdrawing burns stETH and follows Lido and Ethereum withdrawal queues, so timing can vary.

How is web3 staking different from running a validator?

Running an Ethereum validator requires validator setup and operations; Lido batches user deposits through protocol validators and issues stETH to users.
